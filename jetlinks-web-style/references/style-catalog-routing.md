# 业务任务路由与方案输出

## 业务任务 → 方案路由

进入页面后用户的第一任务决定方案优先级。先做业务任务归类，再从对应方案组里挑 1 个推荐方案和最多 3 个替代方案。

| 第一任务 | 推荐候选方案（按优先级） | 说明 |
| --- | --- | --- |
| 浏览定位一批对象 | 资产卡片台账页 / 筛选工作台页 / 标准管理表格页 | 看对象状态识别度与字段密度选择 |
| 横向对比多个对象 | 多对象对比页 / 资产卡片台账页 | 多于 2 个对象需要同视图比较时优先对比页 |
| 处置状态事件 | 处置工作台 / 筛选工作台页 / 时间线日志流 | 告警、工单、审批、巡检 |
| 持续维护单个对象 | 对象详情工作区 / 配置态-运行态切换页 / 主从详情工作区 | 看对象数量与运行态/编辑态分离需求 |
| 配置开通新能力 | 配置向导 / 资源选择器 | 分阶段配置或选资源后再开通 |
| 选择资源 / 模型 / 模板 | 资源选择器 / 配置向导 | 需要预览、筛选、对比时优先选择器 |
| 监控整体态势 | 监控感知页 / 分析钻取页 | 看是看"现在怎样"还是"为什么这样" |
| 分析指标与维度 | 分析钻取页 / 监控感知页 | 多维下钻、对比分析 |
| 查看事件演进 / 操作历史 | 时间线日志流页 / 主从详情工作区 | 需要按时序理解状态变化 |

> 同一业务任务在不同对象规模、状态复杂度、字段密度下会落到不同方案；推荐方案和替代方案必须说明业务差异，不是只换名字。
> `标准管理表格页` 不是信息不足时的兜底。只有第一任务明确是批量检索、列对比或轻量 CRUD 时才推荐；否则优先选择更贴近业务的台账、工作台、详情、处置、时间线或向导方案。

## 方案统一字段

每个方案按下面字段输出，缺哪项都会在落地时漂移：

```
- 业务任务
- 适用场景
- 不适用
- 首屏结构（ASCII 线框骨架）
- 信息密度目标
- 编辑触发梯度
- 核心组件（来自当前 workspace 的 jetlinks-web-core）
- 状态 / 标签锚点
- 主要动作位置
- 不借鉴清单
- 与相近方案的分界
```


## 方案档案输出

方案被用户选定或被默认采用后，给后续实现保留一份简短方案档案：

- solutionName
- primaryFilterSurface
- contentSurface
- detailCarrier
- editMode
- coreComponents
- actionPlacement
- densityTarget
- sidebarMode
- confirmationMode
- rejectedAlternatives
- searchShellDecision

这份档案只锁定业务结构和交互路径，不代表组件已经可用；落地时仍必须回到 `jetlinks-web` 核验当前 workspace 的 `jetlinks-web-core` 导出、组件契约和相邻页面用法。

`searchShellDecision` 默认写 `ConditionFilter`。只有窄改旧页、用户明确要求旧表格风格，或筛选只是少量固定字段且没有路由回显、远程枚举 / 引用选项、保存搜索需求时，才写 `ProSearch` 并说明例外理由。

## 方案选择时的推荐措辞

推荐固定用下面格式：

1. 推荐：`资产卡片台账页`
   - 适合：快速识别对象状态、标签和主动作
   - 结构：顶部条件筛选 + 可选快捷筛选 + 卡片台账
   - 组件：`ConditionFilter` + `ResponsiveGrid` + `EntityCard` / `CardBox`
2. 备选：`筛选工作台页`
   - 适合：高频组合条件定位复杂记录
   - 结构：顶部条件筛选 + 左侧快捷筛选 + 摘要列表
3. 备选：`处置工作台`
   - 适合：状态分栏 + 连续动作流（告警 / 工单 / 巡检）
4. 备选：`对象详情工作区`
   - 适合：围绕单个对象持续查看、局部编辑和配置

如果用户没有明确偏好，给出默认项并直接说明原因。事实清楚且只有一个可信推荐时可记录"默认采用推荐方案"并继续；关键事实缺失、结构风险高，或两个以上方案同样合理时必须先让用户选择。
