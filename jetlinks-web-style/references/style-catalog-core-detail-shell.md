### 7. 详情页头部摘要区：基本信息直接 inline 编辑

所有承载"单对象详情"的方案（对象详情工作区 / 主从详情工作区 / 配置态-运行态切换页，及任何带"对象摘要"的派生方案），头部摘要区必须按下面规范实现，保持跨页面一致。

#### 必须包含的要素（按顺序）

1. **对象身份**：图标 + 名称（**inline 可改**），名称是强主标题
2. **状态**：状态徽标 / chip（用 `j-badge-status` / `MetaChip`，不用裸文字；状态切换通过快捷动作按钮，不放进编辑表单里）
3. **标签**：胶囊集合（用 `ChipGroup` / `AppTag`），**就近支持 + / × 增删**，禁止进编辑表单改标签
4. **说明 / 描述**：副标题位（**inline 可改**），长文本截断 + tooltip 或"展开 / 收起"
5. **归属**：项目 / 租户 / 责任人 / 区域 / 产品等（图标 + 文字，必要时可跳转），通常只读
6. **元信息**：创建时间 / 更新时间 / 版本号（次要文字，弱化展示）
7. **快捷动作**：1-3 个核心动作（启用 / 停用 / 部署 / 同步 / 重启 / 删除等），按对象类型差异化

#### 字段类型 → 编辑方式映射

| 字段 | 编辑方式 | 组件 |
| --- | --- | --- |
| 名称 | hover 显示编辑入口，点击 inline 编辑 | `InputEditable` / `Editable` |
| 说明 / 描述 / 备注 | 同上 | `InputEditable` / `Editable` / `FormItemEditable` |
| 标签 | 就近 + 添加 / × 删除 | `ChipGroup` / `AppTag` |
| 状态 | 通过快捷动作按钮切换，附必要的二次确认 | `j-badge-status` + 动作按钮 |
| 归属 | 通常只读；如允许改，用单字段选择器 inline | `FormItemEditable` + 选择器 |
| 元信息 | 只读 | 文本 |

#### 用户行为细节（跨页面统一）

- hover 字段显示编辑入口（铅笔图标 / 下划线 / 浅色背景），让用户知道这里可改
- 点击字段直接进入 inline 编辑态
- Enter 保存、Esc 取消、失焦保存（具体按工作区统一约定）
- 校验失败就近气泡 / 红框提示，不跳出弹窗
- 保存成功用 toast 或区域提示，文案带对象名（例：`已更新设备 A 的名称`）
- 必填字段为空时回退到上一个有效值，并提示原因

#### 跨页面一致性

- 摘要区在 Tabs 切换时**始终保持可见**，不能随 Tab 内容滚走或重复展示
- 多个详情页（设备 / 产品 / 模型 / 通知配置 / AI 应用等）的头部摘要区结构必须复用同一套组件与节奏，避免每个详情页自造一套排版
- 快捷动作按钮的位置（右上角）、数量（1-3 + 更多）、命名风格在不同对象间保持一致

#### 反模式（明确禁止）

- ❌ 在头部摘要区右上角放一个"编辑"按钮，点击弹出包含名称 / 说明 / 标签等基本信息的编辑表单
- ❌ 头部只展示纯文本"名称：xxx"，没有 hover 显示的编辑入口
- ❌ 标签的增删必须进入编辑弹窗才能完成
- ❌ 把状态作为表单字段提供"切换状态"下拉，而不是用快捷动作
- ❌ 把基本信息（名称 / 说明 / 标签 / 状态）和复杂配置混在一个编辑表单里一起改
- ❌ 不同详情页的头部摘要区结构、字段顺序、动作位置都不一样
- ❌ 头部摘要区跟随 Tab 切换消失，或者每个 Tab 内重复一遍摘要

#### ASCII 骨架（统一参考）

```
┌────────────────────────────────────────────────────────────────────────────┐
│ [图标] 对象名（inline）           ● 状态徽标   [启用][停用][更多 ▾]       │
│ 说明（inline，长文截断 + tooltip / 展开）                                 │
│ 标签：[标签 1 ×] [标签 2 ×] [+ 添加]   归属：项目 A · 责任人 张三 · 区域 X│
│ 创建 2024-01-01 · 更新 2026-05-13 · 版本 v1.2                              │
└────────────────────────────────────────────────────────────────────────────┘
```

### 8. 编辑交互样式统一

避免"功能 A 用 chip 改标签，功能 B 用 select 改标签"、"功能 A 失焦保存，功能 B 必须点保存按钮"这种碎片化。整站任一字段类型的编辑控件、触发方式、保存反馈必须一致，由本节锁定。

#### 字段类型 → 编辑控件唯一映射

| 字段类型 | 唯一推荐控件 | 触发方式 | 保存方式 |
| --- | --- | --- | --- |
| 名称 / 标题 | `InputEditable` / `Editable` | hover 显铅笔 / 下划线 → 点击进 inline | Enter 保存 + 失焦保存 + Esc 取消 |
| 说明 / 描述 / 备注 / 注释 | `InputEditable` / `Editable` / `FormItemEditable` | 同上 | 同上 |
| 标签 | `ChipGroup` / `AppTag` | 就近 + / × 按钮 | 即时提交 |
| 状态 | `j-badge-status` / `MetaChip` + 快捷动作按钮 | 快捷动作 + 必要的二次确认 | 立即生效 |
| 布尔开关 | `a-switch` 或工作区 Switch 包装 | 直接切换 + 必要的二次确认 | 立即生效 |
| 单选 / 多选 / 枚举 | `a-select` / `a-radio` / `a-checkbox` 或 `FormItemEditable` 内嵌 | inline 选择器 | 选择即提交 / Enter 提交 |
| 时间 / 日期 / 范围 | `a-date-picker` / `a-range-picker` | inline 选择 | 选择即提交 |
| 数字 | `a-input-number`（带单位用 `Input` + `addonAfter`） | inline 编辑 | Enter / 失焦保存 |
| 富文本 / Markdown / JSON / 代码 | 工作区统一编辑器（不要每页换一种） | 进 `SectionCard` 编辑态或抽屉 | `StickyActionBar` 保存 |
| 资源关联（设备 / 产品 / 用户 / 组织等） | 工作区统一资源选择器 | inline 选择器 / 选资源弹层 | 选择即提交 |
| 跨字段强校验 / 阶段化 / 破坏性 | `EditDialog` / `JlDrawerShell` / 配置向导 | 显式动作 | 表单 OK 提交 |

> 同一字段类型禁止跨模块换控件；业务有差异（例如某模块"名称"必须唯一并需即时去重）时，仅扩展校验，不更换控件。

#### 通用编辑动作节奏（跨页面统一）

- **触发**：hover 字段显示编辑入口（铅笔图标 / 浅色背景 / 下划线选其一，整站统一一种），点击进入编辑态
- **键盘**：Enter 保存 / Esc 取消 / Tab 切换下一字段
- **失焦**：默认失焦保存；只有"破坏性 / 不可恢复"字段允许失焦不保存，并显式给保存 / 取消按钮
- **校验**：就近气泡 / 红框，校验失败回退到上一个有效值，不跳弹窗
- **加载态**：保存中行内小型 loading（spinner / 灰化），不阻塞整页
- **成功反馈**：toast 或区域提示，文案带对象名与影响范围（例：`已更新设备 A 的名称`、`已为产品 B 添加标签"测试"`）
- **错误反馈**：保留用户输入 + 错误气泡，附"重试 / 撤销"
- **二次确认**：仅用于"启用 / 停用 / 删除 / 重置 / 解绑 / 切换运行模式"等破坏性 / 不可逆动作；轻量字段不要叠加 Popconfirm

#### 跨模块一致性

- 同名字段（名称 / 标签 / 状态 / 说明 / 备注 等）在不同业务模块下，控件、触发、保存反馈完全一致
- 编辑入口的视觉表达（铅笔位置 / 颜色 / hover 反馈）整站一种，避免一个模块铅笔在右、另一个模块铅笔在左
- 同类破坏性动作的二次确认文案、按钮顺序、危险色用法在整站一致

#### 反模式（明确禁止）

- ❌ 在 `jetlinks-web-core` 已经有 `InputEditable` / `Editable` / `ChipGroup` 等时，业务模块自造可编辑文本 / chip / 行内表单 / 校验气泡
- ❌ 名称在一个页面 inline，在另一个页面要点"编辑"按钮弹表单
- ❌ 一个页面 Enter 保存，另一个页面只能点保存按钮
- ❌ 一个页面失焦保存，另一个页面失焦丢弃
- ❌ 标签在 A 模块就近 + / ×，在 B 模块必须打开"编辑标签"弹窗
- ❌ 同一字段类型在不同模块用不同控件（例：状态有的地方用 `Switch`、有的地方用 `Select`、有的地方用纯按钮）
- ❌ 成功反馈一个模块用 toast、另一个模块用页面 alert、另一个完全没反馈
- ❌ inline 编辑场景再叠加"编辑 / 保存 / 取消"三按钮 + 独立表单壳
- ❌ 用 `a-input` 直接覆盖文本展示（没有 hover 编辑入口、没有保存反馈）

#### 工作区落地参考

- 编辑控件优先源：`jetlinks-web-core/src/components/index.ts` 的 `InputEditable` / `Editable` / `FormItemEditable` / `ChipGroup` / `AppTag` / `j-badge-status` / `MetaChip` / `StickyActionBar` / `SectionCard` 等
- 实现前在相邻同域模块（设备 / 产品 / 模型 / 通知 / 规则 / AI 应用 / 项目 / 组织等）找一个已经按本节落地的页面作为视觉与节奏锚点；本模块未实现的控件不要自造，先核查 `jetlinks-web-core`，确实缺失时按 `$jetlinks-conventions` 的根因路径反馈

### 详情页 10 条硬规则

所有对象详情类页面（对象详情工作区 / 主从详情工作区 / 配置态-运行态切换页 等）落地时必须按这 10 条走。它跟"详情页头部摘要区"是同一组规范的不同层面：摘要区规范聚焦头部基本信息怎么放、怎么编辑；本节聚焦**整页的分区、字段、密度、空数据处理**。

| # | 规则 |
| --- | --- |
| 1 | 段是**单层卡**（白底 + 1px border + 12px radius + 16-22px padding）；段内字段无 box，**禁止 sub-card 嵌套**（≥ 2 层 border 矩形） |
| 2 | 段头 = 14px icon + H2(13px / 600 weight)，icon 统一主色 / accent，**不每段一种装饰色** |
| 3 | **不要把"非指标"硬做成 KPI 卡板**——分类（"边缘"）、字符串（"ONNX"）、为 0 的数字、相对时间不是指标 |
| 4 | **没数据不渲染**：占位字符串（以"待配置"开头）→ 不渲染该项；段内所有字段都被过滤 → 整段 `v-if` 隐藏，不留空壳 |
| 5 | 段语义对齐：字段不为"填满 grid"硬塞到不属于的段（例：「I/O 契约」段不要塞「负责人 / 运行位置」） |
| 6 | Hero 卡 = 品类 icon block（40-44px，品类色 15% 透明度做底）+ 标题行（**最多 3 元素**：name + 状态徽标 + 1 个类型 chip）+ 描述（max-width 720）+ inline `·` 分隔 meta 行 |
| 7 | 高密度优先：段间 `gap: 14px`、段内 padding ≤ 22px、行高 1.6-1.65；**整页纵向高度不能比改造前更长** |
| 8 | 段内允许的层次（按需出现）：段头 → 引言 → 主体（KV / 列表 / 单行）→ 底部软 CTA |
| 9 | 单字段占位 → 转化为段底软 CTA（例：`+ 补充 X →`），**不静默吃掉**该字段 |
| 10 | 大屏（≥ 1100px）允许短小段卡两两并排，长段（含引言 / CTA / 大字段）独占整行；中屏自动 collapse 单栏 |

**详情页 "AI 味" 自检（7 条不要，PR 前对照）**：

- ❌ sub-card 嵌套（≥ 2 层 border 矩形）
- ❌ 每段头一种装饰色 icon
- ❌ 把"非指标"做成 KPI 板
- ❌ 让占位文案占满版面
- ❌ 彩色装饰条 / blob / 大引号 / 单侧彩色线条
- ❌ hero 标题行塞 ≥ 4 个 pill
- ❌ 让内容贴左对齐而右半灰底空白

**关联实体段（反向引用模式）**：资产类详情页展示反向引用（"被哪些智能体 / 项目 / 规则使用"）时，**不放右栏 sticky**，做成主区独立段：

- 段头 = icon + H2 + 数量徽章（灰底圆角 pill 显示数字）
- 引言 = 一句话说明"这些实体怎么用本资产"
- 主体 = `auto-fill` 卡片网格，每张卡片是整块可点链接
- **没有反向引用时整段 `v-if` 不渲染**（rule 4）
- 用 composable / service 实时反查，不依赖 mock 字段静态值

> 设计依据：HuggingFace / ModelScope / Notion 都把"used by"放主区段，不放右栏；issue / PR 类工作流详情才适合右栏 sticky。

### 侧栏与导航交互

**侧栏 active 态按嵌套深度分两档**：

- **A 档 · 浅嵌套侧栏**（6-8 个一级链接，无 / 浅子菜单，例如应用中心 / 开发中心一级导航）：保持**去染色 + 文字加粗**的克制风格
  - 文字：本地 ink-1 + `font-weight: 500`
  - icon：本地 ink-1
  - background：transparent
- **B 档 · 深嵌套侧栏**（7+ 模块 × 多子菜单 + 用户长时间停留同一上下文反复跳转，例如项目工作区）：允许 **Linear 风左 2px accent 短条 + 弱底 + accent icon**
  - 文字：本地 ink-1 + 500
  - icon：accent
  - background：弱底
  - 左侧 `::before` 2px accent 短条 + `border-radius: 0 2px 2px 0`

> 分档理由：浅嵌套场景去染色信号足够；深嵌套需要更明确的"我在哪"信号。B 档**不是**"彩色选中块"传统 Material 风——bg 是弱底灰，accent 染色仅在 2px 短条 + icon 上，整体仍克制。

**侧栏折叠分组（8 条精美规则）**：父子分组应统一走共享组件（如 `JlSidebarGroup`），不要每处自己实现：

| # | 规则 |
| --- | --- |
| 1 | caret 用 `chevron-down`，**12px**，最弱色阶（`--text-disabled` 或等价）；折叠时 `rotate(-90deg)`，过渡 `0.15s ease` |
| 2 | 父项跟普通导航链接同样的 28px 行高 / `padding 0 8px` / 小圆角 |
| 3 | 父项**自身没有 active 状态**；子项任一 active 时父项**加粗**，caret 不变色 |
| 4 | 子项区用 **CSS Grid `0fr ↔ 1fr` + `min-height: 0; overflow: hidden`** 做任意高度动画（不要 `max-height: 1000px` 这种 hack） |
| 5 | 整行可点 toggle，不只是 caret 区 |
| 6 | 持久化：传 `persist-key` 时写入 localStorage；不传则每次刷新回 `defaultOpen` |
| 7 | a11y：父项 `aria-expanded` + `aria-controls` 同步；子项区 `role="group"` |
| 8 | 子项 `padding-left` 与父项 icon 视觉对齐（caret 12 + gap 4 + 余 12 = 28） |

**侧栏折叠态**：当整侧栏从展开宽度（如 236）收起到 icon-only 宽度（如 56），**不要嵌**折叠分组组件——用 `v-if / v-else` 切换"分组 vs 平铺"两种渲染。

**顶级路由不渲染 PageHead**：站点顶级 tab 的 root route（工作台 / 应用中心 / 开发中心 / 文档中心首页）**不渲染 PageHead**——标题信息已在顶栏 nav active 态表达；面包屑只有自指自身没有导航价值；强渲染会和 Hero / 主区视觉重复挤压内容空间。非顶级路由（详情页 / 子路由）继续按声明式 API 渲染 PageHead。

**Z-index 层级 token 化**：全局浮动元素（FAB / 浮动面板 / overlay）必须用 token 化的 z-index，**禁裸 `z-index: 9999` / `z-index: 1000`**。推荐层级（升序）：业务模态 (1000) → 自建抽屉 (1100) → toast (1200) → dev tools 浮层 (9999，必须在最上层保证 dev 期间打开模态仍能触达 dev 工具)。

**右下角浮动 FAB 约定**：

- 默认位置 `right: 20px; bottom: 20px`
- 圆形 44×44px，比业务卡稍重的阴影 token
- **单页只允许 1 个 FAB**；如出现多个，按"同位向左 56px / 向上 56px"协议堆叠
- 红点 badge：右上 -3/-3 偏移，最小 18×18，2px 描边
- 展开 panel：360 × 520，三段布局 head / body / foot
- 关闭策略：×按钮 / ESC / 再点 FAB；**点 panel 外不关闭**（避免误触丢状态）
- 窄视口（< 480px）降级为底部抽屉
