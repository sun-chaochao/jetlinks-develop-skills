# JetLinks 工作区结构与能力发现指南

本文件用于帮助智能体在陌生的 JetLinks 系脚手架中快速识别模块边界、聚合结构和能力落点。

它不是静态模块列表。任何模块清单都必须从当前工作区实时发现。

## 何时使用

- 不知道当前工作区有哪些模块
- 不确定功能属于哪个业务域
- 不知道新代码该放在 `modules`、`components`、`microservices`、`apps`、`bootstrap` 还是其他目录
- 不确定某个能力是业务模块、基础组件还是聚合模块
- 当前仓库几乎是空模板，想知道第一批模块该如何落位

## 发现顺序

1. 先看根目录
   - 查找 `pom.xml`、聚合目录、基础组件目录、业务模块目录、资源目录。
   - 常见目录名包括但不限于：`modules`、`components`、`jetlinks-components`、`microservices`、`services`、`apps`、`bootstrap`。
   - 同时检查这些目录下是否存在符号链接；链接进来的目录也应视为当前工作区可用模块。

2. 再看聚合 `pom.xml`
   - 从根 `pom.xml` 和一级聚合模块中找 `<modules>`。
   - 识别哪些目录是聚合模块，哪些目录是真正承载代码的实现模块。

3. 看相邻模块命名与职责
   - 通过目录名、`artifactId`、包名、资源路径判断每个模块负责什么能力。
   - 先找与目标领域最接近的模块，再决定是扩展它还是新建模块。

4. 看模块内部结构
   - 观察是否存在 `entity`、`service`、`web`、`command`、`configuration`、`events`、`resources/i18n` 等目录。
   - 观察是否存在 `core`、`manager`、`api`、`starter`、`plugin`、`adapter` 等分层。
   - 如果存在 `manager` / `core` 分层，通常 `manager` 承载 CRUD、Controller、应用 Service、持久化、权限、i18n 和运行时装配；`core` 承载公共 domain、DTO、命令 / 事件定义、常量、SPI / 扩展接口等契约。

5. 识别软链接目标
   - 如果目录是符号链接，需要同时确认：
   - 链接入口路径：当前项目是通过哪个路径引用它的。
   - 真实目标路径：代码实际存放在哪里。
   - 搜索和读取时，不要因为工具默认行为而漏掉链接目标。
   - 如果搜索工具默认不跟随目录软链接，显式启用跟随链接的搜索方式，或分别检查入口路径和真实路径。

## 模块识别方法

### 低上下文模板仓库处理

如果当前仓库缺少可参考业务模块：

- 先以根目录、聚合 `pom.xml`、已有空目录和命名习惯推断模块放置方式。
- 如果连同类模块都没有，优先创建“最小业务模块”，而不是等待完整参考样例。
- 这时输出里要明确：哪些结论来自现有工作区，哪些结论来自通用约定。

### 判断模块职责

- 如果模块主要提供实体、服务、控制器，通常是业务实现模块。
- 如果模块主要提供工具、公共能力、基础设施，通常是组件或基础能力模块。
- 如果目录下只有 `pom.xml` 和子模块声明，通常是聚合模块。
- 如果模块只暴露 DTO、常量、接口、命令契约，通常是对外契约模块；在 JetLinks manager/core 分层下优先识别为 `core`，不要默认推导为额外的 `api` 模块。

### 判断代码应落到哪里

优先顺序：
1. 目标业务域已有实现模块
2. 同类功能的相邻模块
3. 明确需要新建模块时，再创建新模块

不要因为找不到静态列表就直接新建目录。
如果候选模块是软链接模块，仍然按现有模块处理，而不是重复创建本地副本。

## 智能体操作规范

1. 先给出模块地图，再动代码
   - 至少说明候选模块路径、职责、为什么选它。

2. 不把“当前仓库快照”写进答案或提示词
   - 提示词只保留发现方法，不保留具体模块名表。

3. 用相邻示例确认风格
   - 进入目标模块后，继续查看现有 Entity、Service、Controller、Command、资源文件。

4. 软链接模块要双路径说明
   - 分析时同时说明“项目内入口路径”和“真实目标路径”，避免后续搜索、修改或引用时混淆。
   - 对外说明和代码落点优先使用项目内入口路径；在需要排查搜索遗漏时再补充真实路径。

## 输出模板

当需要先分析时，输出以下四项：

1. 当前工作区的主要结构
2. 与需求最相关的候选模块
3. 推荐落点
4. 是否需要新建模块，以及理由
5. 如果涉及软链接模块，补充其入口路径和真实路径
